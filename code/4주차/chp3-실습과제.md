# 데이터 요구 분석 명세서

## 1. 여행상품 관리
- 여행상품 번호를 관리한다.
- 여행상품명, 출발지, 도착지, 프로그램 정보, 시작 일시, 종료 일시를 관리한다.
- 최소 출발 인원, 최대 인원, 여행 경비를 관리한다.
- 여행상품의 출발 여부를 관리한다.
- 하나의 여행상품은 1명의 직원이 담당하고, 1명의 직원은 여러 여행상품을 담당할 수 있다.
- 여행상품의 출발이 결정되면 관광버스 1대와 운전기사 1명을 배정한다.

## 2. 예약 관리
- 고객이 여행상품을 예약한 날짜를 관리한다.
- 예약금 납부 여부를 관리한다.
- 고객이 예약 후 24시간 이내에 예약금을 결제하지 않으면 예약을 취소한다.
- 여행상품의 출발이 결정된 경우, 1주일 이내에 나머지 여행경비를 결제하지 않으면 예약이 취소되고 예약금은 반환하지 않는다.
- 여행상품의 출발이 취소된 경우, 1주일 이내에 고객에게 예약금을 반환한다.

## 3. 고객 관리
- 고객 id, 이름, 주민등록번호, 휴대폰 번호, 주소를 관리한다.
- 고객 등급은 일반, 우수, 최우수로 구분하여 관리한다.
- 여행상품에 3~5회 참여한 고객은 우수, 6회 이상 참여한 고객은 최우수, 나머지는 일반 등급으로 관리한다.

## 4. 직원 관리
- 직원의 사번, 이름, 주민등록번호, 휴대폰 번호, 급여, 담당 업무, 입사일을 관리한다.
- 직원의 담당 업무는 여행상품 관리, 예약 관리, 관광버스 배차, 직원 관리, 고객 관리로 구분한다.
- 각 직원은 자신이 담당하는 여행상품과 예약 상황을 확인할 수 있어야 한다.
- 현재 약 30명의 직원이 근무한다.

## 5. 관광버스 관리
- 관광버스의 차량 번호, 좌석 수, 출고 연도를 관리한다.

## 6. 운전기사 관리
- 운전기사의 기사 id, 이름, 주민등록번호, 연락처, 운전 경력, 사급, 계약일, 계약 기간을 관리한다.
- 운전기사는 계약직이며 월 단위로 계약한다.

# 트랜잭션 요구 분석 명세서

## 1. 여행상품 관리
- 직원은 자신이 담당하는 여행상품을 검색한다.
- 직원은 새로운 여행상품 정보를 등록한다.
- 직원은 기존 여행상품 정보를 수정한다.
- 직원은 여행상품의 출발 여부를 결정한다.
- 출발이 결정된 여행상품에 관광버스와 운전기사를 배정한다.

## 2. 예약 관리
- 고객이 여행상품 예약 정보를 입력한다.
- 고객의 예약금 납부 여부를 등록한다.
- 예약 후 24시간 이내 예약금이 미납된 예약 정보를 취소 처리한다.
- 출발 결정 후 1주일 이내 잔금이 미납된 예약 정보를 취소 처리한다.
- 출발 취소 시 고객 예약금 환불 정보를 처리한다.
- 직원은 현재 판매 중인 여행상품의 예약 현황을 조회한다.

## 3. 고객 관리
- 고객 정보를 등록한다.
- 고객 정보를 조회한다.
- 고객 정보를 수정한다.
- 탈퇴한 고객 정보를 삭제한다.
- 고객의 여행 참여 횟수에 따라 고객 등급을 갱신한다.

## 4. 직원 관리
- 직원 정보를 등록한다.
- 직원 정보를 조회한다.
- 직원 정보를 수정한다.
- 직원 정보를 삭제한다.
- 직원은 자신의 휴대폰 번호를 변경한다.
- 직원의 사번은 관리자만 변경할 수 있다.

## 5. 관광버스 관리
- 관광버스 정보를 등록한다.
- 관광버스 정보를 조회한다.
- 관광버스 배차 정보를 관리한다.

## 6. 운전기사 관리
- 운전기사 정보를 등록한다.
- 운전기사 정보를 조회한다.
- 운전기사 배정 정보를 관리한다.
